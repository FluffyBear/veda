REST <---> Tarantool exchange protocol
Пакеты запросов к Tarantool имееют общий вид и состоят из трёх элементов:
1. 4 байта — размер msgpack.
2. 1 байт — тип операции.
3. Msgpack соответствующего размера с информацией.

Для операции PUT msgpack состоит из массива, содержащего флаг need_auth,
отображающего необходимость авторизации. Далее идёт user_id равный 
строке с именем пользователя, если need_auth равен true, и nil иначе. 
последовательность индивидов для записи (create, update):
[need_auth, user_id, individual_1, ..., individual_n].

Ответ на запрос PUT представляется размером сообщения (4 байта) и 
masgpack'ом из массива, который содержит общий код выполнения операции 
и коды результатов операции вставки: [general_code, code_1, ..., code_n].
В случае если общий код не является кодом успеха (200), то msgpack состоит
только из него: [general_code].

Для операции GET msgpack состоит из массива, содержащего user_id и 
последовательность uri индивидов для чтения: 
[need_auth, user_id, uri_1, ..., uri_n].

Ответ на запрос GET представляется размером сообщения (4 байта) и 
masgpack'ом из массива, который содержит общий код выполнения операции, 
далее идёт последовательность кодов результатов для чтения, если код чтения 
соответствует успеху (200), то после кода идёт msgpack индивида, если код 
соответствует провалу (XXX), то после него идёт nil:
[general_code, success_code_1, msgpack_1, fail_code_2, nil, success_code_3, ....].
В случае если общий код не является кодом успеха (200), то msgpack состоит
только из него: [general_code].
